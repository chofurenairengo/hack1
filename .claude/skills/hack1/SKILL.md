# Hack-1 / Date My Mate JP — プロジェクト Skill

## 1. プロダクト一行サマリ

**友人が友人のシングルをスライドで紹介する**ライブイベント運営プラットフォーム。connpass / Luma 的ポジショニング。海外で急成長中の「Date My Mate」を日本初上陸させる。

- マッチングアプリではない (スワイプなし)
- インストール不要の Web アプリ
- 母数依存せず**リアルイベントの場の力**で出会いを創出
- テーマ「小さくなる日本」への回答: **消滅した「お節介文化」をテクノロジーで復活**

## 2. 審査コンセプト: Move Hands. Move Hearts.

- **Move Hands** = ユーザに気づかれないこだわり・実装の丁寧さ
- **Move Hearts** = 友人のために手を動かすことが、誰かの心を動かす
- 30 秒ピッチで Hearts、展示会で Hands を見せる

## 3. 技術アピール 3 軸 (Move Hands の体現)

### 3.1 SMI アルゴリズム自前実装と可視化

- `matching` ライブラリに依存せず **Stable Marriage with Incomplete Lists (SMI)** を自前実装
- **男女比均等化をハード制約**に追加した拡張版を開発
- マッチング過程をノード / エッジでリアルタイムアニメーション可視化
- **後悔総和最小化 (Min-Regret Sum)** 実装で数学的公平性を体感させる
- 審査員に「ノーベル経済学賞理論 (Gale-Shapley) の拡張実装」として訴求

### 3.2 マルチエージェント AI によるスライド生成

GPT-4o-mini を 4 エージェントで協調動作させる:

| エージェント | 役割 |
|-----------|------|
| 構成 | 全体構成 (オープニング・本論・クロージング) を設計 |
| コピーライター | 各スライドの文言を最適化 |
| 批評 | 公序良俗違反・個人情報リスクを検知 |
| デザイナー | レイアウトとビジュアル要素を調整 |

単一プロンプトでは届かない品質と安全性を役割分担で実現する。

### 3.3 堅牢性 (エッジケース対応)

- **男女比動的リバランシング**: ドタキャンで比率が崩れても SMI ハード制約をリアルタイム再計算
- **ネットワーク断絶時フォールバック**: Realtime が切れてもローカルで進行できる offline-first
- **不完全データ対応**: 投票しない人がいてもアルゴリズムが破綻しない設計
- 審査員の「ネットワーク切ってみていい？」に耐えるのが「ちゃんと使えるもの」の証

## 4. 開催形式: オンラインコア型ハイブリッド

デモデーが 30 秒ピッチ + 展示会形式のため**オンラインをコア**に据える。オフライン対応は最小差分 (QR 受付 + 会場プロジェクター配信) で実現。

| 機能 | オンライン | オフライン |
|----|--------|--------|
| AI スライド生成 / 画面同期 / 投票 / SMI | ○ | ○ |
| プレゼン表示 | 画面共有 | 会場プロジェクター |
| ビデオ通話 | ○ (交流タイム) | — |
| QR 入場 | — | ○ |

## 5. 参加枠 (3 種類)

| 枠 | 内容 | マッチング |
|----|------|--------|
| プレゼン枠 | 紹介者 + 被紹介者のペア | 被紹介者のみ可 |
| シングル参加枠 | 登壇なしで参加 | 可 |
| オーディエンス枠 | 視聴のみ | 不可 |

オーディエンス枠は初心者の受け皿・次回プレゼン者への転換ファネル。

## 6. データモデル (抜粋)

| テーブル | 主要カラム | 備考 |
|--------|--------|------|
| Users | gender, preferred_genders 他 | LGBTQ+ 前提の設計 |
| Events | mode (online/offline), host_id | モードで UI 分岐 |
| Pairs | presenter_id, presented_user_id | 紹介者と被紹介者 |
| Recommendations | presenter_id, to_user_id | **UI 表示専用**。SMI 入力には使わない |
| Votes | voter_id, round, choices_json | **これが SMI の唯一の入力** |
| Matches | user_a_id, user_b_id, status | SMI の結果 |

## 7. 重要な設計判断

- **紹介者がマッチング相手を選ぶ案は不採用**。被紹介者の自己決定権を守る。代替は「推薦フラグ (星マーク・参考情報のみ)」
- **プレゼン中はビデオ OFF**。集中度を上げる
- **アワードは「おもしろい」「盛り上がった」「友情」の 3 種のみ**。「異性から人気◯位」は**空気を悪くするため絶対に設けない**
- 安全性は**マイナンバー等の技術認証ではなく、友人ペア登壇 = 社会関係資本**で担保する
