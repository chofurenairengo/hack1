## 1. 技術アピール3軸（Move Hands の体現）

審査コンセプト「Move Hands」（ユーザーには気づかれないこだわり・実装の丁寧さ・ディテール）を満たすため、**LLMのAPI呼び出し＋1ライブラリ使用で終わらない「学生離れした技術的深度」**を以下3軸で示す。

### 1.1 SMIアルゴリズム自前実装と可視化

Python の `matching` ライブラリに依存せず、**Stable Marriage with Incomplete Lists（SMI）** アルゴリズムを自前実装。さらに **男女比均等化をハード制約として加えた拡張版**を開発。

- マッチング過程をノード/エッジでリアルタイムアニメーション可視化
- **後悔総和最小化（Min-Regret Sum）** を実装し、数学的公平性を体感させる
- 審査員には「ノーベル経済学賞理論の応用」として紹介

### 1.2 マルチエージェントAIによるスライド生成

単一のLLM呼び出しではなく、GPT-4o-mini を4エージェントで協調動作させる。マルチエージェントシステムは現在ホットなトピックであり、AI時代のものづくりという文脈とも整合。

| エージェント | 役割 |
|------------|------|
| 構成 | 全体構成（オープニング・本論・クロージング）を設計 |
| コピーライター | 各スライドの文言を最適化 |
| 批評 | 公序良俗違反・個人情報リスクを検知 |
| デザイナー | レイアウトとビジュアル要素を調整 |

単一プロンプトでは実現できない品質と安全性を、エージェント間の役割分担で実現する。

### 1.3 堅牢性：エッジケース対応

展示会で審査員にどんな状況を作られても動く設計。

- **男女比の動的リバランシング**：ドタキャンで比率が崩れてもSMIの制約をリアルタイム調整
- **ネットワーク断絶時のフォールバック**：Realtime接続が切れてもローカルで進行できる offline-first 設計
- **不完全データへの対応**：投票しない人がいても破綻しないアルゴリズム設計

→ 「ネットワークを切ってみていい？」と言われても動かせるのが「ちゃんと使えるもの」の証明。

## 2. 技術スタック

### 2.1 フロントエンド（Web PWA）

| 要素 | 技術選定 | 理由 |
|------|---------|------|
| フレームワーク | Next.js (App Router) + React | デザイナー0名でも shadcn/ui で高品質UI構築可能 |
| UI/スタイリング | Tailwind CSS + shadcn/ui + framer-motion | アニメーションでディテールを磨き、Move Hands の体現 |
| 状態管理 | Zustand | 軽量で WebSocket イベントとの連携が容易 |
| ビデオ通話 | Daily.co または LiveKit SDK | WebRTCゼロ実装を避け、数日で統合可能。交流タイムのみ使用 |

### 2.2 バックエンド・インフラ

| 要素 | 技術選定 | 理由 |
|------|---------|------|
| API/DB | Supabase (PostgreSQL) | 認証・Row Level Security をフルマネージド。無料プランで十分 |
| リアルタイム通信 | Supabase Realtime | 同時接続200・メッセージ100/秒。デモに十分 |
| AIスライド生成 | OpenAI GPT-4o-mini × 4エージェント | マルチエージェント協調で品質最適化 |
| マッチング | Python 自前実装 SMI 拡張版 | 男女比均等化制約付き。ライブラリを避けて技術的深さを確保 |

### 2.3 スライドのレンダリング方式

**.pptx ではなく Webコンポーネント（HTML/CSS）** としてレンダリング。

- 会場プロジェクターにブラウザ経由で投影可能
- リアルタイム編集も同じ仕組みで実現
- オンライン/オフライン両モードで共通の実装

## 3. データベーススキーマ

### 主要エンティティ

| テーブル | 主要カラム | 備考 |
|---------|-----------|------|
| Users | id, email, name, age, interests, area, icon_url, has_dating_experience, **gender**, **preferred_genders** | gender/preferred_genders はマッチング用属性（後述） |
| Events | id, title, date, status, host_id, **mode（online/offline）**, venue_id | mode によりハイブリッド切替 |
| EventTickets | id, event_id, max_presenter, max_single, max_audience, price_per_tier | 枠3種類の管理 |
| Participants | id, event_id, user_id, tier, status | tier: presenter / single / audience |
| Pairs | id, event_id, presenter_id, presented_user_id | 紹介者（presenter）と被紹介者の組 |
| Slides | id, pair_id, template_id, content_json | Webコンポーネントとしてレンダリング |
| Recommendations | id, event_id, presenter_id, from_user_id, to_user_id | 紹介者の推薦フラグ（星マーク） |
| Votes | id, event_id, voter_id, round, choices_json | 各0〜3人の投票データ |
| Matches | id, event_id, user_a_id, user_b_id, status | SMIアルゴリズムの結果 |

### 設計上の留意点
- **Recommendations は SMI の入力には使わない**（UI表示のみ）。Votes テーブルのみが入力
- **Events.mode** で online / offline を切替、フロントの挙動を分岐
- Row Level Security でイベント参加者以外はデータ参照不可

### マッチング用属性（gender / preferred_genders）
SMI の男女比均等化制約および相互希望のマッチングに使用するフィールド。LGBTQ+ 対応を前提に以下方針で扱う。

| フィールド | 型 | 値の例 | 用途 |
|-----------|----|------|------|
| `users.gender` | enum | `female` / `male` / `non_binary` / `prefer_not_to_say` | 比率制約の対象カテゴリ（自己申告） |
| `users.preferred_genders` | enum配列 | `['female']` / `['male','non_binary']` / `['any']` | マッチング希望の対象（複数選択可・`any` で制約なし） |

- **未回答（`prefer_not_to_say`）の扱い**：均等化ハード制約の対象外とし、ソフト制約として余剰枠に割り当てる
- **非二元（`non_binary`）の扱い**：`preferred_genders` で自分を望む相手のみとマッチング候補を構成。比率均等化は `female` / `male` の2カテゴリに対して適用し、`non_binary` は別プールで処理
- **イベント単位のポリシー**：`Events` に `matching_policy`（将来拡張）を持たせ、「二元制約あり／制約なし」を切替可能にする設計余地を残す

## 4. 開催モード別の機能差分

| 機能 | オンライン | オフライン | 共通度 |
|------|-----------|-----------|--------|
| AIスライド生成 | ○ | ○ | 100% |
| WebSocket画面同期 | ○ | ○ | 100% |
| 投票システム | ○ | ○ | 100% |
| SMIマッチング | ○ | ○ | 100% |
| 振り分け通知 | ブレイクアウト割り振り | テーブル番号表示 | ほぼ100% |
| プレゼン表示 | 画面共有 | 会場プロジェクター（ブラウザ表示） | 実は同じ |
| ビデオ通話 | ○（交流タイム） | — | オンラインのみ |
| QR入場 | — | ○ | オフラインのみ |

→ **オンライン版を作れば、オフライン対応に必要な追加実装は QR受付と会場画面への配信切替のみ。**

## 7. デモで見せる技術ポイント

### 展示会ブースでの想定シナリオ
1. 審査員がスマホでQR読み込み
2. 事前録画のダミーペアプレゼンを視聴（WebSocketで画面同期される様子を見せる）
3. 投票 → SMIアルゴリズム可視化（ノード/エッジのアニメーション）
4. 「あなたは○番テーブルです」と同期表示
5. 「ネットワーク切ってみます」→ offline-first で継続動作を実演

### 審査員に刺す技術ワード
- ノーベル経済学賞理論（Gale-Shapley）の拡張実装
- 男女比均等化のハード制約付き SMI
- 後悔総和最小化（Min-Regret Sum）
- マルチエージェント協調AI
- offline-first architecture
- WebSocket による全員画面同期
