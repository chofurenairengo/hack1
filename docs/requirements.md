## 2. プロダクト概要

海外で急拡大中の「**Date My Mate**（友人がスライドでシングルの友人をプレゼンするライブイベント）」を日本初上陸させる、**イベント運営Webプラットフォーム**。connpass / Luma のようなポジショニング。

- マッチングアプリではない（スワイプなし）
- インストール不要の Web アプリ
- 母数依存せず、リアルイベントの「場の力」で出会いを創出

### テーマへの接続

「小さくならないためにつくる」に該当。少子化の根本原因「出会いの減少」に対し、**消滅した「お節介文化」をテクノロジーで復活**させる。

## 3. 開催形式：オンラインコア型ハイブリッド

デモデーが30秒ピッチ＋展示会形式であるため、オンラインをコアに据える。

### オンラインをコアにする理由
- 審査員がその場でQRを読み、プレゼン視聴→投票→マッチング結果受取までを一連で体験可能
- オフライン主導だと動画やスクショしか見せられず「体験できないデモ」になる
- Move Hearts の体現として展示会ではオンライン版が圧倒的に強い

### オフライン対応は最小差分で実現
| 差分項目 | 対応 |
|---------|------|
| 入場 | QRコード読み取り追加 |
| スライド表示 | 会場プロジェクターへ切替（参加者が下向かず一体感を生む） |
| ビデオ通話 | 不要（物理的にテーブルに集まる） |
| ドリンク1杯特典 | 会場（飲食店）側が送客特典として負担 |

## 4. 参加枠（3種類）

| 参加枠 | 内容 | マッチング | 想定料金 |
|--------|------|-----------|----------|
| プレゼン枠 | 紹介者＋被紹介者のペア | 被紹介者のみ可 | 通常 |
| シングル参加枠 | 登壇なしで参加 | 可 | 通常 |
| オーディエンス枠 | 視聴のみ | 不可 | 無料〜格安 |

### オーディエンス枠を設ける意義
1. 初めてで怖い層の受け皿（雰囲気だけ体験）
2. 海外既存イベントの構造的欠陥（90%が受動的観客）の修正
3. 次回のプレゼン者候補への転換ファネル

## 5. ユーザー体験フロー

### Phase 1：事前準備

- **アカウント登録**：メール・名前・年齢・趣味・居住地。アイコンは風景やペット等なんでも可（リアル対面前提のため写真スクリーニング不要）
- **紹介者バッジ（任意）**：「恋愛経験あり」を自己申告で表示可能。必須化しない
- **イベント作成**：主催者がターゲット層・参加枠別の人数・男女比を設定、共有リンクを発行
- **エントリー**：プレゼン枠は友人ペアで登録、他は個人で可

### Phase 2：スライド作成（3層構造）

紹介者が被紹介者の魅力を伝えるスライドを以下の3層で作成：

1. **テンプレート選択** — デザイン格差を吸収
2. **AI補助入力** — 「最高なところは？」「笑える弱点は？」等に答えるだけで下書き生成
3. **自由編集** — 紹介者の想いが反映される余地を残す

#### 必須サブ機能
- **被紹介者の事前プレビュー＋NG機能**：日本の文化的配慮として必須
- **不適切表現の自動フィルタ**：公序良俗違反をAI側で検知

### Phase 3：イベント当日

スタンバイ（ビデオOFF）→ ピッチ（全員の画面同期・ビデオOFFで集中）→ 投票（0〜3人を優先順位付き秘密送信、オーディエンスは投票不可）→ マッチング → 「○番テーブルへ」案内 → ブレイクアウト交流（ビデオON）→ 2〜3回繰り返し → 最終投票。

### Phase 4：イベント後

24時間以内に最終投票締切 → マッチング確定 → 双方が「デートに行く」を承諾した段階で連絡先開示。

## 6. 重要な設計判断

### 紹介者の「推薦フラグ」
「紹介者がマッチング相手を選ぶ」案は **不採用**。理由：
- 被紹介者の自己決定権の侵害
- 「初心な子は騙される」前提は20〜30代に対して過保護
- SMIアルゴリズムのフェアネスが崩れる

#### 代替案「推薦フラグ」の仕様
- 被紹介者のマッチング画面に星マークで推薦表示
- 紹介者は最大3名まで推薦可能
- **アルゴリズム入力には含めず、参考情報に留める**

→ 「友人の応援を活用しながらも最終決定権は本人に残す」バランス。お節介文化の復活であり、お見合い強制ではない。

### プレゼン中のビデオOFF
全員が発表スライドに注目すべき時間。感情的負担を減らし、集中度を高める。

### ゲーミフィケーション：アワード設計
- 「おもしろいで賞」「盛り上がったで賞」「友情が深まったで賞」の3種
- **「異性から人気1位」系は空気を悪くするため絶対に設けない**

### 友人ペア参加による安全性
自分の名前と顔を出してステージ上で友人を紹介する構造自体が最強のフィルター。マイナンバー等の技術認証ではなく**社会関係資本による信頼担保**。

## 7. ビジネスモデル

**極限までシンプル：参加費の○％のみ。** 飲食店との金銭やり取りなし。

| 項目 | 内容 |
|------|------|
| プレゼン枠・シングル枠 | 数百円（ランチ以下） |
| オーディエンス枠 | 無料〜格安 |
| ドリンク1杯無料 | 飲食店側が送客特典として負担 |
| 飲食代 | 参加者が店に直接支払い |

飲食店はアイドルタイム（平日夜・昼）に会場登録することで自動集客を得る。Timeleft（仏発・MRR €1M超、日本で400店舗提携）で実証済みの飲食代分離型モデル。

### 拡張性
- 季節イベント連動（バレンタイン・クリスマス）
- オンライン視聴枠の充実（地方在住者向け）
- B2B展開（リファラル採用イベントへの横展開）
- LGBTQ+対応（海外では既に開催実績あり）

### MVPスコープ外
決済システム・マイナンバー連携・飲食店ダッシュボード・オンライン視聴配信・信用ランク。

## 8. 社会的インパクト

### 「お節介文化」の復活
かつて出会いの1位だった「友人紹介」は30.8%（2015年）→10.9%（2024年）に急落、マッチングアプリ（32.2%）と完全逆転。しかし9割が「アプリ疲れ」、月平均18時間の「恋活ロストタイム」が発生。

本プロダクトは友人の応援 × 安全なリアルイベントで、孤立化する日本社会の**ソーシャルキャピタル再構築**を目指す。

### ターゲットユーザー
大学生〜20/30代社会人。特に：
- マッチングアプリに手を出せなかった層（料金・心理的ハードル）
- アプリ疲れで休止中の層（9割）
- 「お節介」があれば結婚できていた層

**スコープ外**：「友人がいない」人は対象外。パートナーよりも先に友達を作るべきであり、それは別プロダクトの領域。

### 初期ユーザー獲得戦略（Tinder式3段階）
1. 1大学で密度最大化（サークル・ゼミ単位で浸透）
2. 友人ペア参加による K-factor ≥ 1 の構造的バイラル
3. 休暇帰省で全国拡散

## 9. デモデー戦略

### 30秒ピッチ：Move Hearts
「友人のために手を動かすことが、誰かの心を動かす」というストーリー。

### 展示会：Move Hands
審査員のスマホでQR読込 → 事前録画のダミーペアのプレゼン視聴 → 投票 → WebSocket同期で「あなたは○番テーブル」→ マッチングの魔法をその場で体感。「リアルイベントでも同じ」と語ってオフライン展開も訴求。

### 審査員アピール要点
1. **日本初** のポジショニング（海外で急拡大中・国内未上陸）
2. **SMI自前実装＋男女比制約** の技術的深さ
3. 審査員自身が体験できる **体験型デモ**
4. テーマとの一貫性（少子化の根本原因への直接アプローチ）
5. **自己決定権の尊重**（推薦フラグは参考情報に留める倫理設計）
